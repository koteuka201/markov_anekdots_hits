# Отчёт по заданию: Markov-анекдотогенератор
запуск  python markov_generator.py anekdots.txt --jokes 5
## Файлы
- `markov_generator.py` — рабочий скрипт (стандартная библиотека Python).
- `anekdots.txt` — датасет (одна строка = один анекдот).

## Промпты, применённые для генерации кода
Ниже приведены 3 примера промптов, которые использовались для получения начальной версии кода от генеративных моделей (ChatGPT/Copilot/Claude). После получения кода он был вручную доработан и отлажен.

Пример промпта 1:
задача  
С помощью GPT (ChatGPT, Copilot, Claude и др.) сгенерируйте и отладьте программу на Python,  
которая генерирует анекдоты с использованием марковских цепей.

Результат — рабочий скрипт (с консольным интерфейсом), который создаёт новые анекдоты  
на основе датасета текстов.

Используйте датасет анекдотов в формате txt или csv (по одной строке на анекдот).  
Можно взять из открытых источников или составить самостоятельно (5 000+ строк).

Ограничения:
Код должен быть создан генеративной моделью и при необходимости доработан вами.
Разрешено использовать только стандартные библиотеки Python (random, re, collections, argparse и т.п.).
Нельзя использовать готовые марковские библиотеки (markovify, textgenrnn и аналоги).
В отчёте должно быть указано, какие промпты применялись для получения кода.

Что сдаётся:
Рабочий скрипт markov_generator.py;
Датасет анекдотов (anekdots.txt) — можно ссылкой на внешнем диске;
Короткий файл report.md с 2–3 примерами промтов и фрагментом вывода программы.

Критерии приемки:
❌ 0 баллов — код не работает, нарушены ограничения, отсутствует подтверждение использования GPT.
✔️ 1 балл — код запускается, генерирует текст, структура проекта соблюдена, промты приложены.
✔️ 2 балла — можно получить только в случае если генератор генерит действительно хорошие анекдоты.
